## Project Description

This project is a production-grade, highly optimized In-Memory NoSQL Key-Value Storage Engine engineered entirely from scratch using Modern C++. Designed to mimic core architectural workflows of enterprise systems like Redis, this storage engine bypasses all high-level standard container libraries (such as `std::unordered_map`) to provide complete developer transparency over low-level dynamic memory distribution, runtime pointer chaining, and deterministic data layouts. At its core, the system utilizes a custom-built Polynomial Rolling Hashing algorithm to index incoming string keys into a static bucket matrix, executing lookups, storage, and deletions with an efficient average-case time complexity of O(1).

To address the inherent volatility of in-memory databases—where data is lost upon application crashes or system restarts—this engine implements a robust database persistence layer driven by a Write-Ahead Logging (WAL) subsystem. Moving away from heavy, slow plain-text configuration files, the persistence architecture reads and writes data via raw high-performance Binary Serialization (`ios::binary`). This byte-streaming framework reinterpret-casts state configurations directly into binary arrays on disk (`wal_storage.bin`), creating an ultra-fast, data-dense archive that handles atomic logging and executes safe, complete crash-resilient data recovery during boot cycles.

Furthermore, the engine introduces a real-time volatile memory optimization layer through an automated Time-To-Live (TTL) data eviction engine. By actively monitoring runtime Unix timestamps, the garbage collection subsystem evaluates key age and automatically purges expired temporary data from volatile RAM, ensuring memory blocks are continuously recycled. The entire engine infrastructure is strictly managed via C++ Smart Pointers (`std::unique_ptr`). By transitioning away from dangerous raw memory allocation and legacy `new`/`delete` statements, the database maintains absolute resource lifecycle tracking with a strict guarantee of zero dynamic memory leaks under heavy operational throughput. Finally, an interactive command-line interface integrated with structural database telemetry allows developers to call diagnostics (`STATS`), analyze runtime bucket distribution density, track live keys, and monitor active hash collisions.
